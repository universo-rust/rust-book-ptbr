# Recursos de programação orientada a objetos

Programação orientada a objetos (POO) é uma forma de modelar programas. Objetos, como conceito de programação, foram introduzidos na linguagem Simula nos anos 1960. Esses objetos influenciaram a arquitetura de programação de Alan Kay, na qual objetos trocam mensagens entre si. Para descrever essa arquitetura, ele cunhou o termo _programação orientada a objetos_ em 1967. Há muitas definições concorrentes para o que é POO; por algumas delas, Rust é orientada a objetos, e por outras, não. Neste capítulo, vamos explorar algumas características normalmente associadas à programação orientada a objetos e ver como elas se traduzem para Rust idiomático. Depois, mostraremos como implementar em Rust um padrão de design orientado a objetos e discutiremos os tradeoffs entre fazer isso e implementar uma solução aproveitando melhor alguns pontos fortes de Rust.
