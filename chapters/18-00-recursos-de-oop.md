# Recursos de Programação Orientada a Objetos

Programação orientada a objetos (POO) é uma forma de modelar programas. Objetos como conceito programático foram introduzidos na linguagem Simula nos anos 1960. Esses objetos influenciaram a arquitetura de programação de Alan Kay, em que objetos passam mensagens uns aos outros. Para descrever essa arquitetura, ele cunhou o termo _programação orientada a objetos_ em 1967. Muitas definições concorrentes descrevem o que é POO, e por algumas delas Rust é orientada a objetos, por outras não. Neste capítulo, exploraremos certas características comumente consideradas orientadas a objetos e como elas se traduzem em Rust idiomático. Depois mostraremos como implementar um padrão de design orientado a objetos em Rust e discutiremos os trade-offs em relação a uma solução que use outros pontos fortes do Rust.
